## 一、项目概述

### 1.1 项目背景

磁力仪广泛应用于地球物理勘探、空间环境监测、导航定位等领域。目前市面上的磁力仪数据采集软件普遍存在传感器类型支持单一、采样率固定、缺乏正交度校正等问题。本项目旨在开发一套通用的、可扩展的磁力仪数据采集与分析系统，支持多种传感器类型和通信方式，提供完善的数据处理与校正功能。

### 1.2 项目目标

开发一套基于 .NET (WPF) 的桌面应用程序，实现以下核心功能：

1. **多类型传感器适配** — 单轴磁通门、三轴磁通门、双三轴磁通门、质子磁力仪等
2. **多通信方式** — 串口（RS-232/RS-485）、TCP/IP 网络连接
3. **灵活采样率配置** — 支持 0.1Hz ~ 500Hz 及自定义采样率
4. **实时数据采集与曲线绘制** — 多通道实时波形显示
5. **数据运算与处理** — 滤波、统计、频谱分析等
6. **正交度校正** — 三轴磁通门正交度计算与补偿矩阵自动生成
7. **数据存储与回放** — 本地数据库存储，历史数据查询与回放
8. **自定义绘图** — 用户可调整数据区间、坐标轴范围、显示参数等

### 1.3 适用对象

- 地球物理勘探工作者
- 磁力仪设备研发与校准人员
- 空间环境监测站技术人员

### 2.3 传感器类型与数据模型

#### 2.3.1 支持的传感器类型

| 传感器类型 | 通道数 | 输出数据 | 典型采样率 | 特殊处理 |
|-----------|--------|---------|-----------|---------|
| **单轴磁通门** | 1 | 单轴磁场分量 (nT) | 1Hz ~ 500Hz | 无 |
| **三轴磁通门** | 3 | X/Y/Z 三轴分量 (nT) | 1Hz ~ 500Hz | 正交度校正 |
| **双三轴磁通门** | 6 | 两组 X/Y/Z 分量 (nT) | 1Hz ~ 500Hz | 各自正交度校正 + 梯度计算 |
| **质子磁力仪** | 1 | 总场强度 (nT) | 0.1Hz ~ 10Hz | 无 |

#### 2.3.2 采样率配置

系统支持以下预设采样率，同时允许用户自定义：

- **预设值**: 0.1Hz, 0.2Hz, 0.5Hz, 1Hz, 2Hz, 5Hz, 10Hz, 20Hz, 50Hz, 100Hz, 200Hz, 500Hz
- **自定义**: 用户输入任意采样率值（0.01Hz ~ 1000Hz）
- **自适应**: 系统可自动检测设备实际输出速率
- **联动校验**: 采样率与传感器类型联动（如质子磁力仪 ≤ 10Hz）
- **不匹配策略**: 设备实际速率与配置不一致时，支持降采样/插值

### 2.4 正交度校正方案

#### 2.4.1 原理说明

三轴磁通门传感器理想情况下三个敏感轴应严格正交（互成 90°）。但实际制造中，轴间存在微小的非正交角度偏差，导致各轴输出相互耦合、测量精度下降。正交度校正的目的是通过数学方法消除轴间耦合误差。

#### 2.4.2 正交度计算流程

在稳定均匀地磁场环境下，对三轴磁通门传感器进行全方位旋转采集数据，系统自动计算正交度补偿矩阵。

**步骤 1 — 数据采集**

在稳定地磁场环境（如无磁干扰的开阔场地）中，缓慢匀速旋转传感器，使其覆盖尽可能多的方位角，采集足够数量的三分量数据 (Bx, By, Bz)。理想条件下，数据应分布在以原点为中心的椭球面上。

**步骤 2 — 椭球拟合**

利用最小二乘法将采集到的数据拟合为一般椭球方程：

```
a·x² + b·y² + c·z² + d·xy + e·xz + f·yz + g·x + h·y + i·z + j = 0
```

通过拟合得到椭球参数，进而提取：
- **偏移向量 (Offset)**: 椭球中心坐标，表示硬铁干扰
- **变换矩阵**: 将椭球变换为球体的矩阵，包含软铁校正和正交度校正信息

**步骤 3 — 正交度补偿矩阵分解**

从变换矩阵中分解出正交度补偿矩阵 M：

```
M = [m11  m12  m13]
    [m21  m22  m23]
    [m31  m32  m33]
```

其中对角线元素接近 1（各轴灵敏度因子），非对角线元素表示轴间耦合系数（正交度误差）。

**步骤 4 — 校正应用**

```
[Bx_corrected]       ([Bx_raw]   [Offset_x])
[By_corrected] = M × ([By_raw] - [Offset_y])
[Bz_corrected]       ([Bz_raw]   [Offset_z])
```

#### 2.4.3 算法实现要点

1. **椭球拟合算法**: 采用约束最小二乘法（如 Li 的方法或 Merayo 的方法），基于 MathNet.Numerics 实现矩阵运算
2. **奇异值分解 (SVD)**: 用于从拟合矩阵中提取正交度参数
3. **数值稳定性**: 对输入数据预处理（去均值、归一化），避免条件数过大
4. **拟合质量评估**: 输出残差统计，供用户判断校正质量
5. **多配置管理**: 支持保存多组正交度校正参数，按传感器序列号关联
6. **可视化验证**: 校正前后的数据在 2D 投影图（XY/XZ/YZ 平面）上对比显示

#### 2.4.4 参考文献方向

- Merayo J M G et al., "Scalar calibration of vector magnetometers"
- Risbo T et al., "Merlin: heading and attitude determination"
- Foster C C, Elkaim G H, "Extension of a two-step calibration methodology to include nonorthogonal sensor axes"
- 磁通门传感器轴间正交度标定方法相关国内文献

```sql
-- 采集会话表
CREATE TABLE sessions (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL,
    started_at      TEXT NOT NULL,

);

-- 磁力数据表
CREATE TABLE readings (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id      TEXT NOT NULL REFERENCES sessions(id),
    timestamp       TEXT NOT NULL,
    ch0             REAL,                -- 通道0 (单轴:主轴 / 三轴:X)
    ch1             REAL,                -- 通道1 (三轴:Y)
    ch2             REAL,                -- 通道2 (三轴:Z)
    ch3             REAL,                -- 通道3 (双三轴:X2)
    ch4             REAL,                -- 通道4 (双三轴:Y2)
    ch5             REAL,                -- 通道5 (双三轴:Z2)
    total_field     REAL,                -- 总场强度
    is_calibrated   INTEGER DEFAULT 0,
    is_ortho_corrected INTEGER DEFAULT 0
);
CREATE INDEX idx_readings_session_time ON readings(session_id, timestamp);

-- 正交度校正配置表
CREATE TABLE orthogonality_profiles (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL,
    sensor_serial   TEXT,                -- 关联传感器序列号
    created_at      TEXT NOT NULL,
    -- 偏移向量
    offset_x        REAL NOT NULL,
    offset_y        REAL NOT NULL,
    offset_z        REAL NOT NULL,
    -- 3x3 补偿矩阵（行优先存储）
    m00 REAL NOT NULL, m01 REAL NOT NULL, m02 REAL NOT NULL,
    m10 REAL NOT NULL, m11 REAL NOT NULL, m12 REAL NOT NULL,
    m20 REAL NOT NULL, m21 REAL NOT NULL, m22 REAL NOT NULL,
    -- 校正质量指标
    residual_mean   REAL,                -- 残差均值
    residual_std    REAL,                -- 残差标准差
    sample_count    INTEGER,             -- 用于拟合的样本数
    notes           TEXT
);

-- 通用传感器校准配置表
CREATE TABLE calibration_profiles (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL,
    sensor_type     TEXT NOT NULL,
    sensor_serial   TEXT,
    created_at      TEXT NOT NULL,
    offset_values   TEXT NOT NULL,        -- 各通道偏移, JSON 数组
    gain_values     TEXT NOT NULL,        -- 各通道增益, JSON 数组
    notes           TEXT
);

-- 应用设置表
CREATE TABLE settings (
    key             TEXT PRIMARY KEY,
    value           TEXT NOT NULL,
    updated_at      TEXT NOT NULL
);
```

### 模块 A：传感器适配与通信

#### A-1：传感器适配器框架 `P0`
- **描述**: 设计 `ISensorAdapter` 接口及工厂模式，实现各传感器类型适配器
- **输入**: 传感器类型枚举、原始字节流
- **输出**: 标准化 `MagnetometerReading` 对象
- **适配器清单**:

| 编号 | 适配器 | 通道数 | 采样率范围 | 备注 |
|-----|--------|-------|-----------|------|
| A-1a | SingleAxisFluxgateAdapter | 1 | 1~500Hz | 输出单轴磁场分量 |
| A-1b | TriaxialFluxgateAdapter | 3 | 1~500Hz | 输出 X/Y/Z + 计算总场 |
| A-1c | DualTriaxialFluxgateAdapter | 6 | 1~500Hz | 两组三轴 + 梯度计算 |
| A-1d | ProtonMagnetometerAdapter | 1 | 0.1~10Hz | 输出总场强度 |

- **验收**: 单元测试覆盖各适配器的正常/异常输入解析

#### A-2：采样率配置系统 `P0`
- **描述**: 实现灵活的采样率配置机制
- **功能点**:
  - 预设采样率列表：0.1, 0.2, 0.5, 1, 2, 5, 10, 20, 50, 100, 200, 500 Hz
  - 自定义采样率输入（范围 0.01~1000Hz）
  - 采样率与传感器类型联动校验（如质子 ≤ 10Hz）
  - 自动检测设备实际输出速率并显示
  - 采样率不匹配时的降采样/插值策略
- **当前状态**: UI 配置与校验已实现；采样率目前仅用于 UI 显示和会话元数据记录，实际采样节奏由设备硬件决定。降采样/插值功能（SamplingRateAdapter）待后续实现。
- **验收**: 各预设值和自定义值均能正确配置，非法值有提示

#### A-3：串口连接模块 `P0`
- **描述**: 基于 `System.IO.Ports.SerialPort` 实现设备连接
- **功能点**:
  - 自动扫描可用串口列表
  - 可配置参数：波特率（9600/19200/38400/57600/115200/自定义）、数据位（7/8）、停止位（1/1.5/2）、校验位（None/Odd/Even）
  - 连接状态监控（连接/断开/错误）
  - 设备热插拔检测与优雅处理
  - 异步读取，不阻塞 UI 线程
- **验收**: 可连接实际串口设备或串口模拟器并稳定接收数据

#### A-4：TCP 连接模块 `P0`
- **描述**: 基于 `TcpClient` + `NetworkStream` 实现网络连接
- **功能点**:
  - IP 地址 + 端口号配置
  - 自动重连机制（指数退避策略）
  - 连接超时配置
  - 心跳检测（可选）
- **验收**: 可连接 TCP 服务端并稳定接收数据，断线可自动重连

#### A-5：协议解析模块 `P0`
- **描述**: 将原始字节流解析为结构化数据
- **功能点**:

### 模块 C：正交度校正

#### C-1：正交度计算引擎 `P0`
- **描述**: 基于椭球拟合算法，从全方位采集数据中计算正交度补偿矩阵
- **算法步骤**:
  1. 接收三轴磁通门在稳定地磁场下的全方位旋转数据 (N×3 矩阵)
  2. 构建椭球拟合方程的设计矩阵 D（N×9 或 N×10）
  3. 最小二乘求解椭球参数向量
  4. 从椭球参数重构对称矩阵 A 和向量 b：A·x² + b·x + c = 0
  5. 计算椭球中心：center = -A⁻¹·b/2（偏移向量）
  6. 对 A 做 Cholesky 分解或 SVD 分解，得到变换矩阵 T
  7. 归一化 T 使校正后数据的球体半径等于参考场强
  8. 输出：偏移向量 (3×1) + 补偿矩阵 (3×3)
- **实现依赖**: MathNet.Numerics（矩阵运算、SVD、Cholesky）
- **验收**:
  - 已知模拟数据（人工构造的椭球数据）校正后残差 < 0.1%
  - 输出补偿矩阵对角线接近 1，非对角线元素量级合理

#### C-2：正交度校正应用 `P0`
- **描述**: 将计算得到的补偿矩阵应用于数据校正
- **功能点**:
  - 实时校正：数据流经过校正环节后再进入图表和存储
  - 批量校正：对已保存的历史会话数据进行回溯校正

  - 多配置切换：可选择不同的校正参数配置
  - 双三轴磁通门：两组三轴各自独立校正
- **验收**: 校正后总场标准差明显降低（与未校正对比）

#### C-3：正交度校正向导界面 `P0`
- **描述**: 步骤引导式 UI，指导用户完成正交度校正全流程
- **步骤设计**:
  1. **选择传感器** — 从已连接或配置的三轴传感器中选择

## 八、技术约束与约定

1. **编码规范**: 遵循 C# 官方编码规范，命名采用 PascalCase（类/方法）和 camelCase（局部变量）
2. **异步编程**: 所有 I/O 操作使用 async/await，禁止 `.Result` 或 `.Wait()` 阻塞
3. **线程安全**: 数据从后台线程传递到 UI 线程必须使用 Dispatcher 或数据绑定机制
4. **内存管理**: 环形缓冲区限制内存中数据点数量，长时间采集不得内存持续增长
5. **错误处理**: 所有通信操作需 try-catch，异常记录日志并通知用户，不得静默吞异常
6. **可扩展性**: 新增传感器类型只需实现 `ISensorAdapter` + 注册到工厂，无需修改现有代码
