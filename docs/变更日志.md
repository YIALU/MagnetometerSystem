# 项目变更日志

本文件记录项目实施过程中与原计划的偏差及方案变更。每次变更需注明日期、变更内容、变更原因，并同步更新《项目计划与任务书》。

---

## 变更记录

### [2026-03-21] 初始化项目

- **变更**: .NET SDK 版本从计划的 .NET 8 调整为 .NET 9（SDK 9.0.312）
- **原因**: 开发机已安装 .NET 9 SDK，但项目 TargetFramework 仍使用 `net8.0-windows`（LTS 版本），确保稳定性。若后续需要 .NET 9 特性，可升级 TargetFramework。
- **影响范围**: 无功能影响，仅 SDK 工具链版本不同
- **计划文档已更新**: 是 — 第九章"项目信息"补充说明

### [2026-03-21] 通信协议架构重构

- **变更**: 将原有硬编码的协议类型（ASCII_CSV / BINARY_FLOAT 等字符串选择）重构为用户完全可配置的协议系统（ProtocolConfig 模型）
- **原因**: 用户要求通信协议可自由配置并存储，包括帧头、帧尾、校验方式、字段映射（字节偏移、数据类型、通道对应关系、缩放/偏移）等
- **影响范围**:
  - 新增: `ProtocolConfig.cs`（模型）、`ConfigurableAsciiParser.cs`、`ConfigurableBinaryParser.cs`
  - 修改: `ParserFactory.cs`（新增 `Create(ProtocolConfig)` 重载）、`ConnectionViewModel.cs`（使用 ProtocolConfig）、`ConnectionView.xaml`（协议配置 UI）
  - 保留: 原有 `AsciiLineParser` / `BinaryFrameParser` 保持向后兼容
- **计划文档已更新**: 待更新

### [2026-03-21] 串口数据接收修复

- **变更**: 将 `SerialDeviceConnection` 的读取方式从 `BaseStream.ReadAsync` + 后台任务改为 `SerialPort.DataReceived` 事件模式
- **原因**: `BaseStream.ReadAsync` 配合 `CancellationToken` 在 .NET SerialPort 上存在已知兼容性问题，导致无法接收数据
- **影响范围**: `SerialDeviceConnection.cs`
- **计划文档已更新**: 无需更新（实现细节层面的修复）

### [2026-03-21] UI 修复

- **变更**: 按钮文本从"连接"/"断开"改为"开始"/"停止"；DataBits/StopBits ComboBox 从 ComboBoxItem 改为数据绑定；新增 Hex 显示开关和清空按钮
- **原因**: 用户反馈按钮文本不变回原始状态，ComboBox 绑定不正确导致默认值不生效
- **影响范围**: `ConnectionView.xaml`、`ConnectionViewModel.cs`
- **计划文档已更新**: 无需更新

### [2026-03-21] 二进制协议配置重构：帧段式(FrameSegment)替代字节偏移(ByteOffset)

- **变更**: 新增帧段式(FrameSegment)协议配置模式，替代原有的 FieldMapping + ByteOffset 模型。用户从上往下逐段配置帧格式（帧头→长度→数据字段→校验→帧尾），系统自动计算偏移和总帧长。
- **原因**: 原有 ByteOffset 方式要求用户手动计算每个字段的偏移量，不直观且容易出错。帧段式"搭积木"方式更符合用户直觉。
- **影响范围**:
  - 新增: `FrameSegment.cs`（帧段模型，含 SegmentType 枚举、ChecksumAlgorithm 枚举）
  - 修改: `ProtocolConfig.cs`（新增 Segments 列表、UsesSegments 属性、ComputeSegmentOffsets/MigrateFromLegacy 方法、段式预设工厂）
  - 修改: `ConfigurableBinaryParser.cs`（新增段式解析路径，兼容旧 FieldMapping 模式）
  - 修改: `ConnectionViewModel.cs`（新增 ProtocolSegments 集合及增删移动命令）
  - 修改: `ConnectionView.xaml`（Binary 模式 UI 替换为帧段列表编辑器，ASCII 模式不变）
- **兼容性**: 旧版 JSON 配置文件通过 MigrateFromLegacy() 自动迁移为段式格式
- **计划文档已更新**: 是

### [2026-03-21] 帧段 ByteCount 可编辑 + 采样率说明

- **变更**:
  1. 帧段的 ByteCount 字段在 UI 上按类型区分：Header/Tail/LengthField/Checksum/Padding 类型可手动编辑（支持帧头 2/4/5 字节、帧尾 1/2 字节等灵活配置）；DataField 类型由 DataType 自动计算（Float=4, Double=8, Int16=2 等），不可手动修改
  2. FrameSegment 实现 INotifyPropertyChanged，DataType 变更时联动更新 ByteCount
  3. 采样率(SampleRate)保留，补充注释说明当前仅用于 UI 显示和会话元数据记录，降采样/插值(SamplingRateAdapter)尚未实现
- **原因**: 用户要求帧头帧尾长度可调（如帧头 4-5 字节），数据字段字节数应随数据类型自动变化
- **影响范围**:
  - 修改: `FrameSegment.cs`（实现 INotifyPropertyChanged，ByteCount/DataType 属性联动）
  - 修改: `ConnectionView.xaml`（字节列条件可编辑）
  - 修改: `ConnectionViewModel.cs`（监听段 PropertyChanged 同步帧长）
  - 修改: `SensorConfig.cs`（SampleRate 注释补充）
- **计划文档已更新**: 是
